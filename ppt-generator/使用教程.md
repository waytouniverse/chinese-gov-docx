# PPT 生成器 - 使用教程

根据 PPT 大纲，用 AI 逐页生成视觉风格统一的 PPT 图片。支持 **Claude Code**、**OpenClaw (QClaw)**、**OpenAI Codex** 三种 AI 编码工具。

## 效果说明

你只需要提供一份 Markdown 格式的 PPT 大纲，技能就会自动：

1. 生成风格自洽的封面页
2. 用 AI 分析封面提取视觉风格（配色、字体、氛围）
3. 以封面为锚点，逐页生成风格统一的内页
4. 输出一套完整的 PPT 图片文件

**最终产出**：每页一张 16:9、2K 分辨率的 PNG 图片，风格高度统一。

---

## 前置条件

### 准备 API 密钥

本技能需要两个 AI 服务：

| 用途 | 模型 | 说明 |
|------|------|------|
| 风格提取 | gpt-5.5（多模态） | 读取封面图，提取配色/字体/氛围等风格信息 |
| 图片生成 | gpt-image-2 | 根据提示词和参考图生成 PPT 页面 |

你需要准备一个兼容 OpenAI 接口的 API 密钥（支持 gpt-5.5 和 gpt-image-2 模型），并记下对应的 base_url。

## 安装技能

根据你使用的 AI 工具，选择对应的安装方式。三种工具的技能格式完全兼容（都是 `SKILL.md` + `scripts/`），只是安装路径不同。

### Claude Code

#### 方式一：复制到技能目录（推荐）

```bash
cp -r ppt-generator ~/.claude/skills/
```

#### 方式二：在项目中使用

把技能文件夹放在项目目录下，在项目的 `.claude/settings.json` 中指定技能路径。

#### 方式三：符号链接

```bash
ln -s /你的实际路径/ppt-generator ~/.claude/skills/ppt-generator
```

#### 安装后目录结构

```
~/.claude/skills/ppt-generator/
├── SKILL.md              # 技能主文件
└── scripts/
    └── parse_outline.py  # 大纲解析脚本
```

---

### OpenClaw (QClaw)

OpenClaw 是 QClaw 应用的底层 Agent 内核，技能格式与 Claude Code 完全一致，只是存放路径不同。

#### 方式一：复制到用户技能目录（推荐）

```bash
cp -r ppt-generator ~/.agents/skills/
```

> OpenClaw 会从 `~/.agents/skills/` 目录加载用户自定义技能。

#### 方式二：通过配置文件添加额外技能目录

编辑 `~/.qclaw/openclaw.json`，在 `skills.load.extraDirs` 中添加你的技能路径：

```json
{
  "skills": {
    "load": {
      "extraDirs": [
        "/Applications/QClaw.app/Contents/Resources/openclaw/config/skills",
        "~/.agents/skills",
        "/你的自定义技能目录路径"
      ]
    }
  }
}
```

#### 方式三：符号链接

```bash
ln -s /你的实际路径/ppt-generator ~/.agents/skills/ppt-generator
```

#### 安装后目录结构

```
~/.agents/skills/ppt-generator/
├── SKILL.md              # 技能主文件
└── scripts/
    └── parse_outline.py  # 大纲解析脚本
```

#### 验证安装

启动 QClaw 后，在对话中输入 PPT 相关请求（如"帮我生成PPT"），技能应自动触发。如果没有触发，检查：

- `~/.agents/skills/ppt-generator/SKILL.md` 文件是否存在
- `openclaw.json` 中 `skills.load.extraDirs` 是否包含 `~/.agents/skills`

### OpenAI Codex

Codex 同样使用 `SKILL.md` 格式的技能，支持手动安装和从 GitHub 安装。

#### 方式一：复制到技能目录（推荐）

```bash
cp -r ppt-generator ~/.codex/skills/
```

#### 方式二：从 GitHub 安装

如果你把这个技能上传到了 GitHub 仓库，可以通过 Codex 内置的安装器安装：

```bash
python3 scripts/install-skill-from-github.py \
  --url https://github.com/<你的用户名>/<仓库名>/tree/<分支>/ppt-generator
```

#### 方式三：符号链接

```bash
ln -s /你的实际路径/ppt-generator ~/.codex/skills/ppt-generator
```

#### 可选：添加 UI 元数据

Codex 支持额外的 `agents/openai.yaml` 文件，用于在 UI 中展示技能信息。如需添加，在技能目录下创建：

```
ppt-generator/
├── SKILL.md
├── agents/
│   └── openai.yaml       # UI 元数据（可选）
└── scripts/
    └── parse_outline.py
```

`agents/openai.yaml` 内容示例：

```yaml
interface:
  display_name: "PPT 生成器"
  short_description: "根据大纲逐页生成统一风格的PPT图片"
  brand_color: "#3B82F6"
  default_prompt: "Use ppt-generator to create PPT slides from an outline"
dependencies:
  tools: []
policy:
  allow_implicit_invocation: true
```

#### 安装后目录结构

```
~/.codex/skills/ppt-generator/
├── SKILL.md              # 技能主文件
├── agents/               # 可选
│   └── openai.yaml       # UI 元数据
└── scripts/
    └── parse_outline.py  # 大纲解析脚本
```

## 配置 API 密钥

安装完成后，打开 `SKILL.md`，找到 **配置信息** 部分，将占位符替换为你自己的密钥：

```yaml
# 替换前
- base_url: `<YOUR_BASE_URL>`
- api_key: `<YOUR_API_KEY>`

# 替换后（示例）
- base_url: `https://api.your-provider.com/v1`
- api_key: `sk-your-actual-api-key`
```

> **安全提醒**：如果你的技能文件会提交到 Git 仓库，请勿在文件中硬编码密钥，建议通过环境变量或 `.env` 文件注入。

## 准备 PPT 大纲

大纲使用 Markdown 格式编写，以下是推荐的格式规范：

```markdown
# PPT 标题

## 封面页
标题：你的主标题
副标题：你的副标题
汇报人：姓名 / 日期

## 第1页：页面标题
- 要点一
- 要点二
- 要点三

## 第2页：页面标题
- 要点一
- 要点二

## 第3页：页面标题
- 要点一
- 要点二
- 要点三
```

### 大纲编写要点

- **封面页**必须包含标题，可选副标题、汇报人/日期
- 每页用 `## 第N页：标题` 或 `## 封面页` 格式开头
- 每页内容控制在 **3-5 个要点**，过长的内容会被精简
- 如需尾页（Q&A/感谢页），在大纲中明确写出，技能不会自动添加
- 大纲有几页就生成几页，不会增删页面

## 使用方法

在 AI 工具的对话中，直接提供大纲即可触发技能：

### 方式一：直接输入大纲文本

```
帮我生成这个PPT：

# 我的产品调研 PPT大纲

## 封面页
标题：产品调研报告
副标题：市场分析与用户洞察
汇报人：张三 / 2025年5月

## 第1页：市场概况
- 市场规模达 500 亿
- 年增长率 23%
- 头部玩家集中度低

## 第2页：用户画像
- 核心用户 25-35 岁
- 一二线城市占比 68%
- 付费意愿中等偏上
```

### 方式二：提供大纲文件路径

```
帮我把 ~/Desktop/outline.md 做成PPT
```

### 方式三：指定风格

```
帮我生成这个PPT，风格用暗黑科技风：

# ...
```

支持的常见风格描述：
- 暗黑科技风 / 暗黑悬疑风
- 政务蓝 / 政务风
- 商务金 / 高端商务风
- 清新简约风
- 也可以用英文描述，如 `minimalist Japanese design style`

## 工作原理（简要）

```
用户提供大纲
      │
      ▼
  生成封面页 ──→ 保存封面图
      │
      ▼
  AI 分析封面 ──→ 提取 STYLE DESCRIPTION + DESIGN SPEC
      │
      ▼
  逐页生成内页 ──→ 每页传入：封面参考图 + 风格描述 + 页面内容
      │
      ▼
  输出完整 PPT 图片
```

风格一致性靠**双通道锚定**：

1. **封面参考图**（最强）：每页生成都把封面图传给 AI 作为视觉参考
2. **风格文字描述**：从封面提取的配色、字体、氛围信息，写入每页的提示词

## 常见问题

### 生成的 PPT 风格不统一？

- 确保安装了 `gpt-image` 技能（用于图片生成），它支持传入参考图
- 如果没有 `gpt-image` 技能，内页生成将缺少封面参考图锚定，风格容易漂移

### 某一页生成失败了？

- 最常见的原因是提示词过长（>500 词），简化该页内容后重试
- 不要省略封面参考图和风格描述，只精简"内容"部分

### 不想用 gpt-5.5，能用其他模型提取风格吗？

- 可以，只要支持多模态图片输入（能看图）+ 文本输出即可
- 修改 `SKILL.md` 中配置信息的 `model` 字段为你想用的模型

### 能生成 PPTX 文件而不是图片吗？

- 目前只输出 PNG 图片，不支持 PPTX 格式
- 你可以将生成的图片导入 PPT/Keynote，每张图作为一页

### 中文内容显示有问题？

- 提示词中的中文文本会被用英文描述位置（如 `title text '产品概览'`）
- 如果中文渲染异常，尝试在内容前加 `Chinese text:` 明确标注语言

### OpenClaw 中技能没有触发？

- 确认 `~/.agents/skills/ppt-generator/SKILL.md` 存在且 frontmatter 格式正确
- 检查 `~/.qclaw/openclaw.json` 中 `skills.load.extraDirs` 是否包含 `~/.agents/skills`
- 尝试重启 QClaw 应用

### Codex 中技能没有触发？

- 确认 `~/.codex/skills/ppt-generator/SKILL.md` 存在
- 检查 `~/.codex/config.toml` 中是否启用了技能加载
- 可以在对话中直接输入 `/ppt-generator` 手动调用
